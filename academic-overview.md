# Academic 学术研究

本页面面向高校研究者、课题组成员和方法论评审者。

它用于回答三个问题：NarrativeOS 在研究流程中解决什么问题、结果如何复核、如何把结果带入论文与开源协作。

## 推荐阅读顺序

- 第一步：先看 [academic/README.md](academic/README.md) 了解学术分区边界
- 第二步：先看 [academic/trust-methodology.md](academic/trust-methodology.md) 明确 Method Kernel 不可变规则
- 第三步：再看 [academic/research-workflows.md](academic/research-workflows.md) 选择 Scenario Composer profile
- 第四步：通过 [academic/publication-support.md](academic/publication-support.md) 检查审稿式压力测试与发布门禁
- 第五步：进入 [academic/reproducibility-kit.md](academic/reproducibility-kit.md) 与 [academic/10-minute-first-success.md](academic/10-minute-first-success.md) 完成可复核闭环

## 架构对齐说明

- 学术方法规则与 [architecture/system/README.md](architecture/system/README.md) 的分层边界一致
- 解释输出遵循 [architecture/insight-engine/README.md](architecture/insight-engine/README.md) 的“结论 -> 证据 -> 原文”链
- 聚合分析遵循 [architecture/corpus-learning/README.md](architecture/corpus-learning/README.md) 的样本回链约束
