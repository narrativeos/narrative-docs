## 0. 使用方式

1. 以 Epic 为单位建任务组（EPIC-PROTOTYPE-CORE）。
2. 每个页面至少创建 FE、UX、QA 三类工单。
3. 按 D1-D10 节奏推进，不允许高优先级页面延期给低优先级页面让路。

## 1. EPIC 与 Sprint 建议

- Epic: EPIC-PROTOTYPE-CORE
- Sprint: Sprint-1 (D1-D5), Sprint-2 (D6-D10)
- 范围: P01、P03、P04、P05、P06、P07、P10 为核心，P02/P08/P09 为增强

## 3. 页面级工单清单

### P01 项目首页 / Workspace Dashboard

1. 标题: P01-FE-01 首页核心状态卡片实现
- 类型: FE
- 优先级: P0
- 估时: 1d
- 依赖: 数据样本 JSON 可读
- 验收标准: 显示样本量、标注进度、最近导出时间，字段无空值

2. 标题: P01-UX-01 首页信息层级与主按钮布局
- 类型: UX
- 优先级: P0
- 估时: 0.5d
- 依赖: PM 已冻结核心任务路径
- 验收标准: 30 秒内可理解下一步动作，主按钮可见且语义明确

3. 标题: P01-QA-01 首页冒烟检查
- 类型: QA
- 优先级: P0
- 估时: 0.5d
- 依赖: P01-FE-01
- 验收标准: 关键数据字段显示正确，导航入口可达 P04/P06

### P03 书籍详情页 / Book Detail

1. 标题: P03-FE-01 书籍元信息与边界提示组件

- 估时: 1d

- 类型: UX

## 5. Ready/Done 定义

- Ready: 需求字段完整、依赖明确、验收标准可测。
- Done: 功能实现、验收通过、证据链接已回填。

## 6. 最低交付门槛（必须满足）

- [ ] P04-P07 主链路可演示且无阻断。
- [ ] 至少 3 名目标用户完成任务测试。
- [ ] 导出结果经抽检一致。
- [ ] High 风险均有处置动作。
