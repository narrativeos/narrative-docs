# NarrativeOS 底层大脑 — 知识时序治理引擎

## 摘要

面向系统管理员、知识治理负责人与平台运营者，对知识库进行全生命周期管理，确保知识资产的时效性、一致性与可追溯性。

## 目标用户

- 系统管理员
- 知识治理负责人
- 平台运营者
- 合规/审计人员

## 核心价值

**让知识不"过期"，让变更可"追溯"。**

NarrativeOS 底层大脑是整个产品矩阵的基石。它不直接面向终端用户，但它决定了你通过插件端看到的参数是否可信、你通过 Studio 入库的知识是否被正确管理。

---

## 典型使用场景

### 场景一：标准更新自动识别

> 国家发布了 GB/T 12345-2025 替代 2018 版，OS 自动：
> 1. 识别新旧版本的差异条款
> 2. 定位所有引用旧版参数的知识节点（共 47 条）
> 3. 标记受影响的工艺参数为"待复审"
> 4. 向相关用户发送提醒通知

### 场景二：变更影响分析

> 治理负责人在后台查看"切削速度"参数的变更历史：
> - 2024-01：初版入库，来源 GB/T 12345-2018
> - 2025-03：标准更新，上限从 220 调整为 200 m/min
> - 受影响文档：12 份工艺单、3 份检验报告
> - 受影响用户：8 人（过去 30 天引用过该参数）

### 场景三：知识审计追溯

> 合规审计时，一键导出某参数的完整证据链：
> - 谁在何时入库了该知识
> - 基于哪个版本的哪个标准
> - 经过哪几轮人工审核
> - 被哪些用户/文档引用过
> - 何时更新、何时废止

## 核心能力

| 能力 | 说明 |
|------|------|
| 时序管理 | 追踪每条知识的版本、生效时间、废止时间 |
| 变更影响分析 | 标准更新 → 自动定位受影响的参数、文档、用户 |
| 主动提醒 | 过期/变更知识的主动通知机制 |
| 审计追溯 | 谁在何时引用了哪个版本的哪条知识 |
| 知识版本控制 | 每条知识支持多版本并存与回滚 |
| 一致性校验 | 自动检测知识冲突（如同一参数在不同标准中取值矛盾） |

---

## 治理流程

```
知识入库（来自 Studio）
  ↓
[版本登记] → 记录来源、时间、审核人
  ↓
[时序标注] → 生效日期、废止日期、适用条件
  ↓
[一致性检查] → 与现有知识冲突？参数矛盾？
  ↓
[发布上线] → 对插件端可见
  ↓
[持续监控] → 标准更新 → 影响分析 → 主动提醒
  ↓
[审计日志] → 全链路可追溯
```

## 技术说明（简要）

- 知识存储支持时序版本管理（非简单覆盖）
- 变更检测基于标准文档的自动化监控管道
- 影响分析基于知识图谱的关系遍历

详细技术实现见 [Architecture 架构](../../architecture/README.md)。

```

  ↓ 查询/写入

  ↓ 知识入库
```

## 常见问题

**Q：OS 大脑是独立产品还是底层能力？**
A：它是底层引擎，不直接面向终端用户，但所有上层产品的知识质量都依赖它。

**Q：如何保证知识的时效性？**
A：通过标准文档的自动化监控 + 知识节点的时序标注 + 变更影响分析 + 主动提醒机制。

**Q：支持多租户/多领域隔离吗？**
A：支持。不同行业/不同企业的知识域可独立管理，互不干扰。
