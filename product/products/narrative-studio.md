# NarrativeStudio — 专家级知识工作台

## 摘要

面向领域专家与知识工程师，将非结构化标准文档（PDF/图片/扫描件）转化为结构化、可检索、可关联的知识资产。

## 目标用户

- 领域专家（机加工、材料、化工等）
- 知识工程师
- 标准维护人员

## 核心价值

**让非结构化文档"活"起来。**

NarrativeStudio 不是文档管理器，而是知识工厂。它将 PDF 中的文字、表格、图片转化为可查询、可关联、可追溯的结构化知识节点。

---

## 典型使用场景

### 场景一：新标准入库

> 专家将新发布的行业标准 PDF 拖入工作台，系统自动完成：
> 1. OCR 识别（含扫描件）
> 2. 实体提取：材质名称、参数数值、刀具型号、温度范围...
> 3. 关系构建：参数 → 标准条款 → 适用条件
> 4. 自动归档：归入对应知识域，生成知识图谱新节点

### 场景二：知识图谱维护

> 专家在可视化图谱中查看"45# 钢"相关的所有知识节点：
> - 关联的标准条款（5 条）
> - 适用的工艺参数（12 组）
> - 历史变更记录（3 次更新）
> - 可手动补充或修正 AI 提取结果

### 场景三：批量处理

> 一次性导入 50 份标准文档，系统按优先级队列处理：
> - 实时查看处理进度
> - 对低置信度提取结果进行人工审核
> - 审核通过后自动发布到知识库

## 核心能力

| 能力 | 说明 |
|------|------|
| 拖拽上传 | 支持 PDF、图片、扫描件等多格式 |
| OCR 识别 | 高精度文字识别，支持扫描件与手写体 |
| 实体提取 | 自动识别材质、参数、刀具、温度等实体 |
| 关系构建 | 实体 → 标准条款 → 适用条件的自动关联 |
| 知识图谱可视化 | 图形化查看实体、关系、来源链 |
| 人工审核 | 专家对 AI 提取结果进行确认与修正 |
| 批量处理 | 多文档队列处理与进度跟踪 |

---

## 处理流程

```
上传文档
  ↓
[OCR 识别] → 文字/表格/图片分离
  ↓
[实体提取] → 材质、参数、刀具型号、温度...
  ↓
[关系构建] → 实体 ↔ 标准条款 ↔ 适用条件
  ↓
[人工审核] → 专家确认/修正（低置信度项优先）
  ↓
[知识入库] → 写入 NarrativeOS 底层知识库
  ↓
[图谱更新] → 知识图谱自动扩展新节点
```

## 技术说明（简要）

- OCR 引擎支持多语言与混合排版
- 实体提取基于 NarrativeOS 训练的行业领域模型
- 知识图谱存储于 NarrativeOS 底层，支持时序版本管理

详细技术实现见 [Architecture 架构](../../architecture/README.md)。

## 与产品矩阵的关系

```
LibreOffice / Word 插件端
  ↓ 查询/写入
NarrativeStudio（专家维护知识） ← 你在这里
  ↓ 知识入库
```

Studio 是知识生产的中台，上游支撑插件端消费，下游驱动 OS 大脑治理。

## 常见问题

**Q：支持哪些文档格式？**
A：PDF（含扫描件）、JPG/PNG 图片、Word 文档。未来计划支持更多格式。

**Q：AI 提取的准确率如何？**
A：取决于文档质量与领域。结构化表格准确率较高，复杂排版需人工审核。系统会标注置信度，低置信度项优先推送审核。

**Q：处理速度？**
A：单份标准文档（20-50 页）通常 1-5 分钟。批量处理按队列排队。
