## 1. 场景概述 | Scenario Overview

机械加工行业拥有海量的标准手册、工艺规范和参数表格。工程师在日常工作中需要频繁查阅这些资料，但传统查阅方式效率低下且容易出错。

### 1.1 典型用户画像

| 角色 | 日常工作 | 痛点 |
|---|---|---|
| 工艺工程师 | 编制加工工艺卡片，确定切削参数 | 手册参数分散，查找耗时 |
| 设计工程师 | 在 CAD 软件中设计零件，需确认材料加工性 | 需切换窗口查手册，打断设计流 |
| 质检工程师 | 检验工件表面粗糙度、尺寸公差 | 需要快速确认标准条款原文 |
| 生产技术员 | 现场调试机床参数 | 需要即时、准确的参数建议 |

### 1.2 核心价值主张

> **在工程师已有的工作环境中，提供基于海量手册知识库的实时参数建议与标准溯源。**

---

## 2. 三大核心优势 | Three Core Advantages

### 2.1 非线性组合检索 | Non-linear Composite Retrieval

**问题：** 工程师在查找加工参数时，往往不是搜一个词，而是搜一个"组合"：

> "材质 45#钢 + 硬质合金刀具 + 粗加工 + 表面粗糙度要求 Ra 3.2"

通用搜索模型面对这种组合查询时，往往只会返回一段通用描述，无法精准定位到手册中的具体表格行。

**NarrativeOS 方案：** 通过结构化知识库将手册中的表格数据拆解为可独立检索的知识原子，支持多维度组合过滤：

```
┌─────────────────────────────────────────────────┐
│  组合检索查询                                    │
│                                                 │
│  材质: [45#钢 ▼]  刀具: [硬质合金 ▼]             │
│  加工类型: [粗加工 ▼]  粗糙度: [Ra 3.2 ▼]        │
│                                                 │
│  [执行查询]                                      │
└─────────────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────────────┐
│  匹配结果：推荐切削参数表                         │
│                                                 │
│  vc: 140-180 m/min                              │
│  fn: 0.15-0.25 mm/rev                            │
│  ap: 2-5 mm                                     │
│                                                 │
│  来源：《机械加工工艺手册》第3版 P.247 表4.2.3    │
│  [点击溯源 →]                                   │
└─────────────────────────────────────────────────┘
```

**能力支撑：**
- Studio 工作台将手册 PDF 中的表格解析为结构化知识条目
- OS 大脑为每条知识维护时序版本和来源链
- 插件端提供组合检索 UI 和实时结果展示

---

### 2.2 溯源即生命 | Evidence Traceability as a Must-have

**问题：** 在制造业中，如果你引用了一个错误的手册参数导致工件报废，这是事故。

| 风险等级 | 场景 | 后果 |
|---|---|---|
| 低 | 参数偏差 5% 以内 | 加工效率略降 |
| 中 | 参数偏差 10-20% | 刀具异常磨损、表面质量不达标 |
| 高 | 引用了已废止的手册版本 | 批量工件报废、安全事故 |

**"证据溯源"在这个场景中的定位：**

> 不是"锦上添花"，而是"刚需"。

**NarrativeOS 方案：** 每条参数建议都可一键溯源到原始手册的具体页码和条款：

```
工程师点击参数建议旁的 [溯源] 按钮：

┌──────────────────────────────────────────────┐
│  证据溯源面板                                 │
│                                              │
│  参数来源：《机械加工工艺手册》第3版           │
│  出版社：机械工业出版社                        │
│  出版年份：2019                               │
│  章节：第4章 切削参数选择                      │
│  页码：P.247                                  │
│  表格：表4.2.3 45钢硬质合金刀具粗加工参数       │
│                                              │
│  [查看原文扫描件 →]                           │
│  [查看该条款变更历史 →]                        │
│                                              │
│  状态：✅ 现行有效                             │
│  最后核验：2026-05-15                          │
└──────────────────────────────────────────────┘
```

**能力支撑：**
- OS 大脑的时序治理引擎追踪手册版本的生效/废止时间
- 变更影响分析：当手册更新时，自动通知引用了旧参数的用户
- 审计追溯：记录谁在何时引用了哪个版本的哪条参数

### 2.3 插件嵌入式体验 | Plugin-Embedded Experience

**问题：** 机械工程师在设计软件（如 SolidWorks、AutoCAD）或简单的工艺表单（Excel/Word）中工作时，绝对不想切换窗口去查手册或问 AI。

**NarrativeOS 方案：** 插件直接嵌入工程师的工作界面，实现"所见即所得的知识支持"。

```
┌──────────────────────────────────────────────────────────┐
│  LibreOffice / Word 工艺卡片编辑界面                       │
│                                                          │
│  工序名称：车削外圆                                      │
│  工件材质：45#钢                                         │
│  刀具选择：硬质合金 CNMN120408                             │
│  切削参数：                                              │
│    vc: [160 ___] ← 光标在此处，右侧悬浮栏自动触发          │
│    fn: [0.20 ___]                                        │
│    ap: [3.0 ___]                                         │
│                                                          │
│                        ┌─────────────────────┐           │
│                        │  智能参数建议         │           │
│                        │                     │           │
│  工程师工作区           │  vc 建议: 140-180    │           │
│  (不离开这里)           │  当前值 160 ✅ 合理   │           │
│                        │                     │           │
│                        │  来源: 工艺手册P.247  │           │
│                        │  [查看溯源] [应用]    │           │
│                        └─────────────────────┘           │
└──────────────────────────────────────────────────────────┘
```

**核心体验原则：**
- **不切换窗口：** 所有知识查询和确认在当前工作界面完成
- **不改变习惯：** 插件是辅助层，不干扰原有的编辑流程
- **选中即查：** 光标停留或选中文本即触发参数建议
- **一键确认：** 建议参数的应用只需一次点击

## 3. 三产品协同工作流 | Three-Product Collaborative Workflow

机械加工手册场景完整贯穿 NarrativeOS 三个产品：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  ① Studio 专家工作台                                │
│     手册专家将 PDF 手册导入 → OCR → 表格解析         │
│     → 结构化知识条目 → 人工审核确认                   │
│                                                     │
└───────────────────────┬─────────────────────────────┘
                        │ 结构化知识入库
                        ▼
┌─────────────────────────────────────────────────────┐
│                                                     │
│  ② OS 底层大脑                                     │
│     维护手册版本时序 → 参数变更影响分析               │
│     → 标准废止主动提醒 → 审计追溯日志                 │
│                                                     │
└───────────────────────┬─────────────────────────────┘
                        │ 知识查询 API
                        ▼
┌─────────────────────────────────────────────────────┐
│                                                     │
│  ③ 插件端 (Word/Excel/SolidWorks)                    │
│     工程师在工艺卡片中填写参数 → 插件实时查询         │
│     → 返回建议值 + 溯源链接 → 工程师一键确认          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 阶段 | 产品 | 操作者 | 产出 |
|---|---|---|---|
| 知识采集 | Studio 工作台 | 手册专家 | 结构化参数条目 |
| 知识治理 | OS 底层大脑 | 系统自动 | 版本链、影响分析、时效提醒 |
| 知识消费 | 插件端 | 工程师 | 工艺卡片、加工程序单 |
